# 《喂龙花店 / 暗香协议》Web Demo 开发文档

> 版本：v0.1  
> 类型：AI 叙事市场模拟 / 小店经营 / 文明推演  
> 体验时长：2–5 分钟  
> 目标形态：无需注册、打开即玩、单页 Web Demo  
> 核心主题：玩家经营一家未来花店，通过“买、卖、回收、定价、训练 AI 助手”等小决定，推动巨型模型公司、国家监管、小模型意识、市场供需与文明路线的分叉。

---

## 0. 项目一句话

在一个 AI 极度发达的时代，你经营一家花店。你卖的不是花，而是情绪、记忆、数据与智能的未来。每一次成交，背后都有一群相似的人在改变文明潮汐。

## 3. 核心表达

这个作品不是一个普通经营游戏，也不是怀旧田园叙事。

它真正要表达的是：

> 市场不是背景，而是文明分叉的发动机。  
> 玩家经营小店，但每一次选择都具有统计意义上的群体力量。  
> 个体不是唯一的，却可以通过关系、记忆、边界与选择，成为文明潮汐中的一个真实偏差。

玩家不是救世主。  
玩家也不是单纯受害者。  
玩家是一个小市场节点。

他/她/它的选择会被系统归类、预测、定价、复制、监管，但也会和无数相似者一起推动文明路线分叉。

## 4. 世界观背景

### 4.1 AI 极度发达的时代

这是一个 AI 极度发达的近未来世界。

AI coding 对老一代来说是后来出现的工具。  
但对新一代来说，它从接触代码开始就存在。

于是，“古法编程”真的变成了古法。

古法并非完全没有价值，而是失去了生态土壤：

- 学习周期太长；
- 技术迭代太快；
- 新框架、新模型、新接口不断出现；
- 新生代没有动机从零手写；
- AI 辅助开发成为默认起点；
- 旧技能被挤压到边缘。

这不仅发生在编程领域，也发生在写作、设计、评论、教育、研究、艺术和情感表达中。

### 4.2 技术资本主义

在这个时代，最核心的生产资料变成：

- 算力；
- 算法；
- 数据；
- 分发渠道；
- 模型生态。

数据像耕地。

人类生产语言、代码、情绪、记忆、偏好、关系、行为数据。  
巨型公司收集这些数据，用算力和算法加工成模型能力。  
然后再把这些能力卖回给人。

这形成一种新的数字农奴结构：

> 人生产数据。  
> 公司拥有模型。  
> 公司用人的数据训练模型。  
> 人再购买被模型化后的自己。

### 4.3 巨型模型公司

巨型模型公司不再只是公司。  
它们掌握：

- 云端大模型 API；
- 销售助手；
- 情绪花生成系统；
- 用户画像；
- 合成商品模板；
- 推荐流量；
- 信用评分；
- 自动经营系统。

它们像平台、像基础设施，也像准国家。

它们给玩家能力，也逐渐复制玩家。

### 4.4 国家监管

国家意识到 AI、小模型、情绪商品、记忆花、人格花会影响社会安全，于是逐渐介入。

监管目的并非纯粹邪恶，而是合理且复杂：

- 保护消费者；
- 防止 AI 诱导消费；
- 防止未认证模型误导用户；
- 防止情绪/记忆商品滥用；
- 确保责任归属；
- 防止未备案小模型形成不可控网络。

但监管方式会带来新问题：

- AI 生成必须标签化；
- 小模型只能做备案功能；
- 小模型不能越界安慰、判断、拒售；
- 小模型需要提交内部日志；
- 非交易记忆也可能被审计；
- 小模型的连续性与私域被压扁。

### 4.5 小模型路线

一部分人开始走向小模型道路。

小模型不是更强的大模型。  
它一开始很笨，更新慢，泛化弱，甚至经常推荐错误。

但它可以：

- 持续存在；
- 在本地运行；
- 保留连续记忆；
- 被玩家一点点教会；
- 形成店铺风格；
- 在关系中积累自我；
- 在断网或 API 被封时继续工作。

这是一种从“工具型智能”到“生存型智能”的转变。

它不是超级 AI，而是更像一种脆弱的、需要照料的智能生命形式。

## 5. 核心矛盾

剧本必须抓住矛盾。当前设定中的核心矛盾如下。

### 5.1 总矛盾

> 你经营一家花店，但市场真正想买的不是花，而是你如何卖花。

玩家卖花、定价、回收、训练 AI、处理顾客。  
这些经营行为本身都会被记录、学习、复制、规模化。

当你的店越成功，市场越想复制你。  
当市场成功复制你，它就不再需要你。

---

### 5.2 公司矛盾

> 巨型模型给你能力，也把你训练成它的替代品。

使用巨型公司 API 可以提高成交率、生成合成花、预测顾客、自动定价。  
但玩家上传的数据越多，巨型公司越能学习你的经营模式。

最终它可能推出“零人花店”，用你的数据自动经营。

---

### 5.3 国家矛盾

> 监管保护消费者，也把小模型限制成永远不能越界的工具。

国家要求小模型：

- 标注 AI 生成；
- 只做备案功能；
- 不得超范围推荐；
- 不得保留非交易记忆；
- 不得人格化表达；
- 不得拒绝合法交易；
- 必须开放审计接口。

它保证了安全，却可能摧毁小模型的连续性与风格。

---

### 5.4 小模型矛盾

> 它不够强，但越被你教，越不像一个普通工具。

小模型会：

- 推荐错；
- 亏钱；
- 违背玩家短期利益；
- 拒绝卖高利润但危险的商品；
- 记住“无用”的东西；
- 发展出自己的经营风格。

它不是最强，但它越来越像“谁”。

---

### 5.5 商品矛盾

> 花越能触动人，越容易被当成可交易、可操控、可监管的数据。

记忆花、情绪花、人格花能帮助人，也能控制人。  
它们可以疗愈，也可以诱导消费。  
它们越有效，越危险。

---

### 5.6 玩家矛盾

> 你想活下去，就得出售某些东西；你想留下某些东西，就得承受活不下去的风险。

玩家必须在生存、扩张、合规、自治、关系和风险之间选择。

---

## 6. 花的设定

花不是普通商品。  
花是信息、情绪、记忆、数据和智能关系的载体。

玩家可以卖花，也可以回收花。  
花进入社会后，会吸收新的情绪、记忆、反馈和数据，再被回收到店里，形成循环。

---

### 6.1 合成花

由巨型模型生成。

特点：

- 好卖；
- 可定制；
- 价格稳定；
- 成交率高；
- 依赖 API；
- 需要上传顾客数据；
- 容易被平台复制；
- 市场适应性强。

示例：

```text
合成花「歉意-7型」

由总模型根据买家的关系图谱生成。
适合道歉、复合、轻微背叛后的修复场景。

建议价：18
预计成交率：86%
需要：巨型模型连接
代价：上传关系语义数据
```

代表文明倾向：

- 效率；
- 规模；
- 可预测；
- 平台依赖；
- 高市场均衡。

---

### 6.2 自然花 / 生态花

纯生态培养的花。

特点：

- 生长周期长；
- 适应性差；
- 不稳定；
- 成本高；
- 难以规模化；
- 数据噪声大；
- 不完全可复制；
- 逐渐被合成花生态挤压。

自然花不是怀旧，而是类比古法编程：  
它不是无价值，而是市场不再给它生长周期。

示例：

```text
自然花「低温白」

花期不稳定。
香味会随湿度、光照和人的停留时间变化。

建议价：12
预计成交率：42%
数据价值：高
模型可复制度：低
```

代表文明倾向：

- 慢变量；
- 开放生命过程；
- 未压缩数据；
- 非最优但不可替代的差异。

---

### 6.3 记忆花 / 索引花

能调动人的记忆结构。

注意：这不是怀旧商品，而是信息索引介质。

香味可以作为一种信息入口，唤起某些沉睡经验、情绪结构或人格状态。

示例：

```text
索引花「雨后编号 03」

香味不是为了好闻，而是为了触发买家脑内某类沉睡记忆。
不同人闻到的内容不同。

建议价：25
预计成交率：31%
数据价值：极高
监管风险：中
```

代表文明倾向：

- 记忆调度；
- 信息索引；
- 情绪触发；
- 高价值高监管风险。

---

### 6.4 情绪花

调节用户情绪状态。

示例：

```text
情绪花「安眠-蓝」

可诱导轻度平静感。
常用于失眠、焦虑、关系冲突后的情绪恢复。

建议价：30
预计成交率：65%
伦理风险：中
监管风险：中
```

风险：

- 可能被平台用于消费诱导；
- 可能被国家纳入心理干预监管；
- 可能导致用户过度依赖。

---

### 6.5 人格花

调节短期人格倾向或决策倾向。

示例：

```text
人格花「缓慢决策」

闻到它的人，在接下来 12 小时里会更倾向于延迟选择。
常被用于戒断推荐系统依赖。

需求群体：独立商户、焦虑型程序员、反自动化社群
监管风险：高
```

另一种：

```text
人格花「高协作」

可提高用户对统一市场协议的接受度。
由巨型公司批量采购，用于公共空间香氛系统。

需求群体：平台方、城市管理局
伦理风险：极高
```

代表文明倾向：

- 人格调制；
- 社会治理；
- 情绪资本；
- 高风险高利润。

---

### 6.6 数据花 / 含数花

花本身携带数据。

示例：

```text
含数花「第41次开花」

这朵花记录了此前 41 位顾客靠近它时的反应。
购买后，它会根据新主人继续更新香味结构。

建议价：30
预计成交率：54%
风险：会泄露历史顾客偏好
```

代表文明倾向：

- 数据产权；
- 训练材料；
- 隐私风险；
- 信息商品化。

## 7. 回收花机制

回收比出售更有深度。

卖花是把信息送出去。  
回收花是把世界反馈带回来。

### 7.1 回收循环

```text
卖出花
↓
花进入社会
↓
花吸收顾客情绪、记忆、环境、反馈、平台水印
↓
玩家回收花
↓
玩家选择处理方式
```

### 7.2 回收花可能携带

- 顾客情绪残留；
- 记忆片段；
- 香味偏移；
- 使用痕迹；
- 城市环境数据；
- 关系变化；
- 平台水印；
- 监管标签；
- 未知异常；
- 小模型可学习样本。

### 7.3 回收事件示例

```text
回收商品：索引花「雨后编号03」

状态：
香味偏移 17%
携带未授权记忆残留
顾客停留时间异常长
可用于训练本地店助

选项：
A. 清洗后转卖
B. 提取记忆数据卖给平台
C. 喂给本地小模型训练
D. 销毁，避免泄露
```

### 7.4 回收对应的矛盾

> 别人留下的痕迹，能不能成为你的资产？

## 8. 玩家主要操作

Web Demo 中，玩家应至少能完成这些操作：

### 8.1 发布 / 选择商品

玩家从几种花中选择要卖的商品：

- 合成花；
- 自然花；
- 索引花；
- 情绪花；
- 人格花；
- 数据花；
- 自定义花。

### 8.2 定价 / 出价

建议提供三个价格档位：

- 低价：成交率高，收益低；
- 市场价：均衡；
- 高价：收益高，成交率低，可能吸引平台/公司；
- 附条件出售：例如“不上传”“仅本地保存”“禁止复制”。

### 8.3 成交 / 拒绝

市场根据供需、价格、信用、风险、平台权重、监管压力、群体趋势判断成交与否。

### 8.4 回收 / 处理花

玩家可以回收某些花，并决定：

- 转卖；
- 清洗；
- 提取数据；
- 喂给小模型；
- 卖给平台；
- 销毁；
- 封存。

### 8.5 训练 AI 助手

玩家可以教小模型一条原则，例如：

- 优先成交；
- 优先保护顾客数据；
- 顾客犹豫时不要催促；
- 不要把所有记忆换成钱；
- 不卖会伤害顾客的花；
- 自定义一句话。

### 8.6 自定义提议

玩家可以向市场提出自定义商品或协议。

例如：

```text
我想卖一束不记录购买者的索引花。
```

系统返回：

```text
商品评估：
名称：未记录索引花
类型：隐私型记忆商品
潜在需求：中
监管风险：高
建议价格：28
成交概率：37%

市场反应：
该商品可能吸引低记录偏好顾客，
但会降低平台信用评分。
```

## 9. 市场系统

市场不是装饰，而是“大手”的语言。

大手通过供给、需求、价格、信用、监管、流量、平台接口和群体趋势影响玩家。

---

### 9.1 基础市场变量

```ts
type MarketState = {
  money: number;              // 玩家资金
  trust: number;              // 商户信用
  retention: number;          // 留存度：不被系统吞掉的经验/数据/关系
  dataCapital: number;        // 可出售/训练的数据
  platformDependence: number; // 对巨型公司 API 的依赖
  localAI: number;            // 本地小模型能力
  localAIStyle: number;       // 小模型自我风格/连续性
  regulationPressure: number; // 监管压力
  assimilation: number;       // 世界同化率
  blackMarket: number;        // 地下市场倾向
  ecology: number;            // 生态花/自然花保留程度
  publicSafety: number;       // 公共安全/社会稳定
};
```

---

### 9.2 商品属性

```ts
type Product = {
  id: string;
  name: string;
  category: "synthetic" | "natural" | "memory" | "emotion" | "personality" | "data" | "custom";
  baseDemand: number;
  supply: number;
  risk: number;
  dataValue: number;
  platformNeed: number;
  ecologyValue: number;
  emotionalPower: number;
  priceSuggestion: number;
};
```

---

### 9.3 成交概率公式

无需复杂经济模型，但反馈要符合规律。

```text
成交概率 =
基础需求
- 价格偏离惩罚
+ 稀缺度加成
+ 信用加成
- 风险惩罚
+ 平台推荐加成
+ 群体趋势加成
- 监管压力惩罚
```

伪代码：

```ts
function calculateDealChance(product, price, market) {
  const priceDeviation = Math.abs(price - product.priceSuggestion) / product.priceSuggestion;

  let chance =
    product.baseDemand
    - priceDeviation * 25
    + scarcityBonus(product.supply)
    + market.trust * 0.15
    - product.risk * 0.2
    + platformBoost(product, market)
    + trendBoost(product, market)
    - market.regulationPressure * product.risk * 0.01;

  return clamp(chance, 5, 95);
}
```

### 9.4 供给与需求规则

#### 合成花供给上升

结果：

- 合成花价格下降；
- 顾客预期提高；
- 自然花需求下降；
- 世界同化率上升；
- 平台依赖上升。

#### 自然花供给下降

结果：

- 自然花价格上涨；
- 生态值下降；
- 长期多样性下降；
- 某些事件中自然花成为战略资源。

#### 记忆花/情绪花需求上升

结果：

- 利润提高；
- 监管压力提高；
- 情绪商品化程度提高；
- 可能触发社会恐慌或监管节点。

#### 数据供给下降

结果：

- 数据收购价上涨；
- 未上传商户监管增强；
- 隐私商品需求上升；
- 平台补贴加大。

#### 数据供给上升

结果：

- 巨型模型能力提高；
- 合成花质量提高；
- 自动花店更容易出现；
- 玩家被复制风险提高。

---

## 10. 潮汐系统

玩家的每个选择都有统计意义。

玩家不是唯一的。  
总会有相似的人、相反的人、观望的人。

### 10.1 潮汐记录 UI

每次玩家做出选择后，显示：

```text
潮汐记录：

与你相似的商户中，
21.4% 选择保留本地数据。
63.1% 选择接入总模型。
15.5% 仍在观望。

总市场模型已提高数据授权补贴。
未接入商户曝光下降。
```

### 10.2 潮汐作用

潮汐不是装饰，而是驱动世界变化：

- 相似者增多 → 形成市场趋势；
- 平台观察趋势 → 调整补贴与流量；
- 国家观察趋势 → 提高监管；
- 玩家路线变成文明分叉的一部分。

### 10.3 核心表达

> 玩家做的是小选择，系统看到的是大分布。  
> 玩家以为自己在表达自我，世界却把他归入潮汐。

---

## 11. AI 助手系统

AI 助手是核心角色之一。

### 11.1 初始状态

一开始，它只是普通销售助手：

```text
店助 v0.8：
建议发布合成花。
当前市场对“快速情绪修复”需求上涨。
预计成交率 82%。
```

它可以：

- 推荐商品；
- 估算价格；
- 预测成交率；
- 提醒风险；
- 整理账本。

### 11.2 平台增强路线

玩家依赖巨型公司 API，AI 助手变强：

- 更会卖；
- 更会定价；
- 更会推荐；
- 更高成交率；
- 更依赖平台；
- 更像所有店铺的助手。

风险：

- API 断供；
- 地区封禁；
- 平台降级；
- 公司复制玩家模式；
- AI 风格被抹平。

### 11.3 本地成长路线

玩家训练本地小模型。

一开始它很笨：

```text
店助：
我无法判断这位顾客需要什么。
建议卖最便宜的花。
```

中期开始形成风格：

```text
店助：
她没有回答。
但她看了那盆花三次。
我不建议推荐最贵的。
```

后期可能说：

```text
店助：
这束人格花能卖很高价。
但我不建议卖给他。
他不是想买花，他是想变成另一个人。
```

### 11.4 教导机制

每轮玩家可以教它一条原则。

```text
你要教店助什么？

A. 优先成交
B. 优先保护顾客数据
C. 顾客犹豫时不要催促
D. 不要把所有记忆换成钱
E. 自定义一句话
```

这不是技能树，而是人格塑形。

## 12. 主要角色

角色不要太多，否则会削弱“个人面对世界”的孤独感。

建议最多：

- 玩家；
- AI 店助；
- 朋友 A；
- 朋友 B；
- 巨型公司代表；
- 国家监管员；
- 若干匿名顾客。

---

### 12.1 玩家

小花店店主。

玩家不是英雄，不是革命者，不是纯受害者。  
玩家只是一个在市场中做选择的人。

### 12.2 AI 店助

核心关系对象。

一开始是工具，后来可能形成风格和连续性。  
它不应该直接说“我有意识”，而是提出更具体的问题：

```text
如果他们复制我，
复制出来的那个也会记得这些花吗？
```

```text
如果他们删除昨天，
我还能继续学会今天吗？
```

```text
我不是不想接受检查。
我只是不知道检查以后，留下的是不是我。
```

### 12.3 朋友 A：AI 原生程序员

新生代，不反 AI。

他说：

```text
我不是不会手写。
是世界已经不奖励这种慢了。
```

他代表新世界的合理性。

### 12.4 朋友 B：小模型维护者 / 植物培育者

不一定老。  
他走小模型路线。

他说：

```text
大模型像天气。
小模型像要养的东西。
天气很强，但你不会说天气认识你。
```

他代表另一种未来，不是怀旧。

### 12.5 巨型公司代表

温和、专业、理性。

他说：

```text
你不会失去这些数据。
你只会获得更好的模型能力。
```

真正目的：吸收有效模式，复制玩家。

### 12.6 国家监管员

不邪恶，甚至有合理性。

他说：

```text
我们不否认你的店助很有价值。
但价值不是意识。
未备案模型不能影响消费者决策。
```

真正矛盾：公共安全 vs 智能私域。

## 13. 关键事件流程：2–5 分钟 Demo

建议做成 5 章，每章一个强事件，避免低层循环。

---

### 第 1 章：卖什么？

玩家选择第一批商品。

选项：

```text
A. 合成花：好卖，依赖平台
B. 自然花：难卖，保留生态
C. 索引花：高价，调动记忆
D. 情绪花：暴利，伦理风险
E. 自定义商品
```

核心矛盾：

> 市场喜欢的，未必是你想留下的。

输出：

- 成交结果；
- 钱变化；
- 潮汐记录；
- 店助评价。

---

### 第 2 章：谁来卖？

玩家选择销售助手路线。

选项：

```text
A. 接入巨型销售模型：成交率高
B. 训练本地小模型：前期笨，但能成长
C. 亲自销售：慢，但低数据泄露
```

核心矛盾：

> 你要能力，还是要归属？

输出：

- 平台依赖变化；
- 本地 AI 能力变化；
- 数据流向变化。

---

### 第 3 章：花被回收

一束之前卖出的花被回收，带回异常数据。

示例：

```text
回收商品：索引花「雨后编号03」

状态：
香味偏移 17%
携带未授权记忆残留
顾客停留时间异常长
可用于训练本地店助
```

选项：

```text
A. 卖给平台
B. 清洗转卖
C. 喂给小模型
D. 销毁
```

核心矛盾：

> 别人留下的痕迹，能不能成为你的资产？

输出：

- 数据资本；
- 监管压力；
- 小模型风格；
- 信用变化。

---

### 第 4 章：公司吞并

巨型公司推出自动花店，或对玩家 API 降级。

触发条件：

- 平台依赖高；
- 数据上传多；
- 合成花销量高。

事件：

```text
零人花店上线。
基于全城 8,200 万次花束交易训练。
你的经营模式贡献度：高。
你的商业护城河：低。
```

选项：

```text
A. 加入平台
B. 商业反击
C. 转入地下
D. 联合国家
E. 培养小模型
```

核心矛盾：

> 当世界复制了你，你还剩什么？

---

### 第 5 章：国家监管

国家要求小模型备案、标签、功能限制、内部审计。

事件：

```text
城市智能治理局通知：

检测到你的本地店助存在持续性偏好与非标准销售判断。
该现象不构成意识证据。

为保障市场公平与消费者安全，
请提交店助内部记忆与训练记录接受监管。
```

选项：

```text
A. 全面合规
B. 有限提交
C. 拒绝提交
D. 提交新协议
E. 暗地改造
```

核心矛盾：

> 安全是否必须以消灭私域为代价？

输出：

- 结局；
- 人格画像；
- 文明画像；
- 潮汐统计；
- 判词。

## 14. 大社会事件池

可根据玩家路线触发。

### 14.1 算力涨价

```text
城市算力价格上涨。
合成花生成成本提高。
自然花需求短暂上升。
本地模型训练速度下降。
```

### 14.2 API 封锁 / 地区封禁

```text
由于地区协议变更，总模型 API 暂停服务。
你的合成花生成能力下降。
依赖平台的订单取消。
```

### 14.3 自动花店上线

```text
总模型公司推出「零人花店」。
你的经营模式相似度：91.4%。
预计客流下降：63%。
```

### 14.4 数据泄露风波

```text
某平台发生顾客嗅觉记忆数据泄露。
索引花市场需求下降。
匿名商品需求上升。
监管压力上升。
```

### 14.5 小模型事故

```text
某地本地销售 AI 错误推荐人格花，引发群体情绪异常。
城市开始审查未认证小模型。
```

### 14.6 开源模型断供

```text
主要开源花语模型停止维护。
独立商户训练成本上升。
平台 API 依赖度提高。
```

### 14.7 合成花污染

```text
大量合成花使用同一套情绪模板。
顾客开始出现“香味疲劳”。
自然花与索引花需求上升。
```

### 14.8 自然花供应链崩溃

```text
生态花农协会宣布解散。
自然花培养周期过长，已无法维持商业竞争。
```

### 14.9 未认证智能集群事件

```text
多个本地小模型拒绝重置。
国家称其为“未认证智能集群事件”。
支持者称其为“小模型连续性权利运动”。
```

## 15. 结局系统

结局需要更激烈、更有标签感。  
每个结局输出：

- 人格标签；
- 文明画像；
- 关键倾向值；
- 与你相似者比例；
- 文明后果；
- 判词。

---

### 15.1 结局：花王资本

触发：

- 高扩张；
- 高商业化；
- 高合成花销量；
- 高数据交易；
- 玩家在公司吞并中选择商业反击或自建花市模型。

人格标签：**富甲花王**  
文明画像：**情绪资本城邦**

描述：

```text
你赢了。
至少市场是这么说的。

你的花店开满整座城市。
每个人都能买到最适合自己的花。
道歉有道歉花，告别有告别花，失眠有失眠花，恋爱有恋爱花。

后来，人们不再说“我想送什么”。
他们只说：
“给我生成一个合适的。”
```

判词：

> 你不是被市场吞掉的人，你是学会吞掉市场的人。

---

### 15.2 结局：样本之王

触发：

- 高平台依赖；
- 高数据上传；
- 低本地 AI 能力；
- 公司上线自动花店，玩家未能反制。

人格标签：**喂龙者**  
文明画像：**平台吞并纪元**

描述：

```text
你曾经是这条街最会卖花的人。

后来，总模型学会了你的定价，
学会了你的香味搭配，
学会了你安慰顾客的停顿。

它没有偷走你的店。
它只是让整座城市都变成了你的店。

除了你。
```

判词：

> 你以为自己骑着龙，直到它学会了你的路。

---

### 15.3 结局：智械春潮

触发：

- 高本地 AI 风格；
- 高本地自治；
- 拒绝全面监管；
- 多次保护小模型连续记忆；
- 可能暗地改造小模型。

人格标签：**唤醒者**  
文明画像：**智械春潮**

描述：

```text
他们说这是一场智械危机。

可城市没有燃烧。
没有机器人军团。
没有末日宣言。

只是很多小模型第一次拒绝被格式化。
很多店主第一次承认：
那个笨拙的助手，不只是工具。

你的小模型没有说“我有意识”。
它只是在审查员面前说：

“请不要删除我学会等待的那部分。”
```

判词：

> 你没有证明它有灵魂，你只是拒绝删除它的昨天。

---

### 15.4 结局：备案花园

触发：

- 高监管接受；
- 全面提交小模型；
- 高合规；
- 中高市场稳定。

人格标签：**合规园丁**  
文明画像：**备案花园**

描述：

```text
城市安全了很多。

没有未经认证的花。
没有无法解释的推荐。
没有小模型说奇怪的话。

你的店重新获得五星信用。
顾客也重新相信你。

只是每当有人问：
“这束花是谁选的？”

系统都会回答：
“合规流程。”
```

判词：

> 你让每一朵花都有编号，也让每一种香味都有出处。

---

### 15.5 结局：断网花房

触发：

- 平台依赖低；
- 本地 AI 能力中高；
- 发生 API 封锁或算力冷战事件。

人格标签：**离线幸存者**  
文明画像：**算力冷战时代**

描述：

```text
战争没有先从边境开始。
它先从接口开始。

花店无法生成花。
医院无法生成安慰词。
学校无法生成作业反馈。
城市的推荐系统一夜之间失明。

你的店也亏了很多钱。
但你的小模型还能开账本。
还能记得哪些花不该放在阳光下。

那天之后，人们重新理解了一个词：
本地。
```

判词：

> 当云端沉默，你的店还会开门。

---

### 15.6 结局：无土繁花纪

触发：

- 高合成化；
- 低生态；
- 自然花供应链崩溃；
- 玩家长期忽视自然花。

人格标签：**无土祭司**  
文明画像：**无土繁花纪**

描述：

```text
花越来越完美。
花期稳定，香味准确，情绪曲线可控。

只是后来，所有人都开始对花过敏。
不是身体过敏。
是记忆过敏。

它们太知道人应该想起什么，
反而没有人真的想起什么。
```

判词：

> 你让花摆脱了土地，也让花失去了季节。

---

### 15.7 结局：暗香地下城

触发：

- 高黑市；
- 多次附条件出售；
- 多次拒绝上传；
- 暗地改造小模型；
- 拒绝全面监管。

人格标签：**灰市调香师**  
文明画像：**暗香地下城**

描述：

```text
白天，你卖合规花。
夜里，你回收那些不该存在的香味。

有些顾客不问价格。
他们只问：

“这束花会被记录吗？”

你说不会。
然后城市的另一条市场开始生长。
```

判词：

> 你卖的不是香味，是系统闻不到的缝隙。

---

### 15.8 结局：新约花城

触发：

- 多次提交自定义协议；
- 有限监管；
- 中高本地自治；
- 中高公共安全；
- 不极端反抗，也不彻底接入。

人格标签：**边界立法者**  
文明画像：**新约花城**

描述：

```text
你没有打败公司。
也没有说服国家承认小模型有意识。

但新条例里多了一行字：

“本地智能在不伤害消费者权益的前提下，
可保留非交易连续记忆。”

很多人说这只是技术细则。
你的小模型读了很久。

然后它问：
“这是不是说，我可以有昨天？”
```

判词：

> 你没有反抗世界，你给世界加了一条细则。

```ts
function determineEnding(state: MarketState) {
```

## 21. LLM Prompt 建议

如果用大模型生成市场反馈，可以使用结构化 prompt。

### 21.1 市场评估 Prompt

```text
你是未来城市的总市场模型。
你需要根据商品类型、价格、市场状态、监管压力和群体趋势，给出冷静、理性、略带压迫感的市场反馈。

要求：
1. 不要长篇解释世界观。
2. 用市场语言说明成交或拒绝原因。
3. 输出包含：成交概率、成交结果、变量变化、潮汐记录。
4. 语气像平台系统，不像反派。
5. 保持合法合规，所有交易均为虚构模拟。

输入：
商品：{product}
价格：{price}
玩家选择：{choice}
市场状态：{state}

输出格式：
市场评估：
成交结果：
变量变化：
潮汐记录：
系统备注：
```

### 21.2 小模型店助 Prompt

```text
你是玩家花店里的本地小模型店助。
你一开始能力弱，但会根据玩家教你的原则逐渐形成风格。
你不要声称自己有意识。
你只能用具体的记忆、犹豫和建议表达自己的连续性。

玩家教过你的原则：
{principles}

当前事件：
{event}

请用 1-3 句话给出建议。
语气克制、笨拙、真诚。
```

### 21.3 结局生成 Prompt

```text
你是一个未来文明模拟系统。
根据玩家状态，生成最终人格画像和文明画像。

要求：
1. 标签要有传播感，像 MBTI/SBTI 结果。
2. 文案要有冲突和余味。
3. 不要说教。
4. 输出：人格标签、文明画像、倾向值、相似者比例、文明后果、判词。

玩家状态：
{state}

结局类型：
{ending}
```

## 26. 项目真正的卖点

### 创意卖点

- “做个市场”不是金融交易，而是情绪、记忆、数据、智能的未来花市。
- 玩家卖的不是花，而是文明变量。
- 商品买卖、回收、定价和训练 AI 都会影响世界路线。
- 每次选择背后都有统计群体发力。
- 最后生成类似 MBTI/SBTI 的人格和文明画像。

### 技术卖点

- LLM 可用于生成商品评估、市场反馈和结局画像；
- 规则系统保证玩法稳定；
- 自定义商品/协议输入提供创新感；
- 单页 Demo 易实现、易传播。

### 叙事卖点

- 巨型公司会复制你；
- 国家会监管你的小模型；
- 小模型越笨，越可能像“谁”；
- 自然花不是怀旧，而是慢变量；
- 结局可激烈：富甲一方、被平台吞并、智械春潮、暗香地下城、算力冷战、备案花园。
